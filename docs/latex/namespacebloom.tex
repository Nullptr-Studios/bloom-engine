\doxysubsection{Detailed Description}
Contains the core classes and functions for the bloom engine. 

The {\ttfamily bloom} namespace encapsulates the main components and utilities required for the bloom engine. This includes the main engine class, window management, factory for object creation, and core utilities.

The {\ttfamily bloom} namespace is designed to provide a comprehensive set of tools for managing the game loop, rendering, and event handling. It includes classes for initializing and managing various subsystems such as the window, renderer, and input handling. The namespace is structured to facilitate the creation, management, and execution of a game using the bloom engine.

Key Classes\+:
\begin{DoxyItemize}
\item {\bfseries{\doxylink{classbloom_1_1_engine}{Engine}\+:}} Manages the main game loop, rendering, and event handling.
\item {\bfseries{\doxylink{classbloom_1_1_window}{Window}\+:}} Manages the window and its events.
\item {\bfseries{\doxylink{classbloom_1_1_factory}{Factory}\+:}} Handles the creation and management of game objects.
\end{DoxyItemize}

Creates an entry point for the Bloom engine. 

This function serves as the primary entry point for the engine, allowing it to handle the main execution flow instead of the game directly. By abstracting the {\ttfamily main} function, the engine can take full control over initialization, execution, and shutdown processes.

\begin{DoxyNote}{Note}
For future Xein\+: This design is intentional to ensure that the engine manages the {\ttfamily main} function rather than the game project. Any game-\/specific logic should integrate with this entry point.
\end{DoxyNote}
While this function is declared here, its implementation resides in \doxylink{main_8cpp}{main.\+cpp} within the game project. This separation ensures modularity between the engine and game layers.
