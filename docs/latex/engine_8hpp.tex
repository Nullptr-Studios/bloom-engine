Creates an entry point for the Bloom engine. 

This function serves as the primary entry point for the engine, allowing it to handle the main execution flow instead of the game directly. By abstracting the {\ttfamily main} function, the engine can take full control over initialization, execution, and shutdown processes.

\begin{DoxyNote}{Note}
For future Xein\+: This design is intentional to ensure that the engine manages the {\ttfamily main} function rather than the game project. Any game-\/specific logic should integrate with this entry point.
\end{DoxyNote}
While this function is declared here, its implementation resides in \doxylink{main_8cpp}{main.\+cpp} within the game project. This separation ensures modularity between the engine and game layers.

\begin{DoxyReturn}{Returns}
A pointer to the {\ttfamily Engine} class instance, which is used to manage the engine\textquotesingle{}s lifecycle. 
\end{DoxyReturn}
