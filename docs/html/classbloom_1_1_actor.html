Detailed Description

Represents a renderable and potentially interactive object within the game world.

The Actor class extends the Object class, adding rendering capabilities. It serves as the base class for any object in the game that needs to be visually represented in the world. Actors manage their own textures, materials, and rendering properties. They participate in the game's update loop through the Begin, Tick, and End methods, allowing for dynamic behavior and animation.

Loads textures for the actor's visual representation.

This method loads the textures used to render the actor. It supports loading albedo, roughness-metallic-occlusion (RMO), normal, and emission textures. If no paths are provided for the textures, placeholder textures will be used instead.

Parameters
descriptorSetLayoutThe descriptor set layout for the textures. This layout defines how the texture data will be accessed by the shaders.
albedoThe path to the albedo texture file. The albedo texture defines the base color of the actor's surface.
rmoThe path to the roughness-metallic-occlusion (RMO) texture file. This texture controls the surface properties of the actor, including how rough or smooth it appears and how metallic it is.
normalThe path to the normal map texture file. The normal map modifies the perceived surface normals of the actor, adding detail and depth without increasing the polygon count.
emissionThe path to the emission texture file. The emission texture defines areas of the actor that emit light, creating a glowing effect.
